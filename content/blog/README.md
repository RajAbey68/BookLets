# content/blog

Every `.md` file here is one blog post. The filename (minus `.md`) is the
slug and the URL: `2026-07-20-my-post.md` renders at `/blog/2026-07-20-my-post`.
`README.md` itself is never treated as a post.

## Frontmatter

```yaml
---
title: "How Double-Entry Bookkeeping Saves STR Hosts at Tax Time"
description: "One-sentence summary shown on the blog index and in search results."
date: "2026-07-20"
tags: ["bookkeeping", "str-hosts"]
keywords: ["short-term rental bookkeeping", "airbnb accounting"]
author: "BookLets Team"
draft: false
---
```

- `draft: true` keeps a post out of the public index, RSS feed, and sitemap,
  and 404s the page outside local development. This is the human-review gate
  for anything generated by `npm run blog:generate` — flip it to `false`
  once you've read the post and are happy with it.
- `tags` and `keywords` are optional (default to `[]`); `author` defaults to
  `"BookLets Team"`.
- Body content is standard GitHub-flavored Markdown.

## Generating a new post

Inside a Claude Code / Cowork session, trigger the `blog-post` or
`content-repurposing` skill under `.claude/skills/` (e.g. `/blog-post write
about ...`). Each writes a `draft: true` file here for you to review. See
`docs/BLOG-SYSTEM.md` for the full workflow.
